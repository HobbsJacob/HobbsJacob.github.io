Campaign Creator

(Drag for more images)

Over the summer of 2018-2019 I interned for Xaxis at their Auckland office. I created a web application using ASP.NET Core and Azure to help automate a section of the advertising campaign creation process. The application used the REST API's available for Salesforce and App Nexus to reduce manual data entry to a minimum.

Advertisement set up information is stored in Salesforce. Previously workers would manually move the data from Salesforce into AppNexus, a platform for creating advertising campaigns.

With this application, the user grants the application access to Salesforce and AppNexus through OAuth. The application then pulls information for advertising campaigns that are due to start in the next n-days from a Salesforce API, and loads it into the page. Users then select which campaigns they wish to set live through AppNexus, as well as a settings template for the campaign. The server then creates JSON representations of the various campaign components and sets them live with the AppNexus API.

Tag QA

Advertising campaigns use pixels to track how an add is interacted with on a web page. Pixels are invisible images which should be loaded whenever a specific action is performed, e.g. the advertisement is loaded or clicked. When a browser makes a request to load a pixel, the server hosting that pixel knows that event took place.

Campaign Managers record these pixels in Excel sheets and must manually check to make sure that each pixel is firing correctly. This WPF application reads the Excel sheets and uses Selenium, a browser automation tool to automatically load and interact with the page on a schedule. The user is allerted if any pixels do not load correctly.
